# HWP COM 수동 검증 체크리스트

- 대상 버전: HwpMate v8.7
- 목적: 단위 테스트로 확인하기 어려운 한컴오피스 한글 COM 자동화, 관리자 권한 GUI, 실제 파일 저장 동작을 배포 전 확인합니다.

## 준비

- Windows 10/11 64-bit
- Python 3.10 이상 또는 `HWP변환기_v8.7.exe`
- 한컴오피스 한글 2018 이상 설치
- 관리자 권한으로 앱 실행
- 테스트용 `.hwp`, `.hwpx` 문서 각 1개 이상
- CLI 보조 검증 시 관리자 권한 PowerShell에서 `python tools/hwp_com_smoke.py --input <샘플.hwp> --format PDF --output-dir <출력폴더>` 실행

## 기본 실행

- [ ] 관리자 권한 실행 시 경고 없이 메인 창이 열린다.
- [ ] 일반 권한 실행 시 관리자 권한 필요 안내가 표시된다.
- [ ] 이미 실행 중인 상태에서 두 번째 앱을 실행하면 "이미 실행 중" 안내 후 종료된다.
- [ ] 파일 모드에서 `.hwp`, `.hwpx` 파일 추가가 가능하다.
- [ ] 폴더 모드에서 하위 폴더 포함 on/off에 따라 미리보기 파일 수가 바뀐다.
- [ ] 관리자 권한 상태에서도 탐색기 드래그 앤 드롭이 동작한다.

## MainWindow 컨트롤러 분리 회귀 확인

- [ ] 테마 전환, 형식 카드 선택, 폴더/파일 모드 전환이 즉시 UI에 반영된다.
- [ ] 파일 모드에서 파일 추가, 선택 제거, 전체 제거 후 상태바 파일 수가 정확히 갱신된다.
- [ ] 폴더 모드에서 형식 변경 또는 하위 폴더 포함 on/off 변경 시 미리보기 스캔이 다시 시작된다.
- [ ] 시스템 트레이 열기/종료 메뉴와 `Ctrl+O`, `Ctrl+Shift+O`, `Ctrl+Enter`, `Esc` 단축키가 동작한다.
- [ ] 변환 중 입력 위젯이 비활성화되고, 취소 또는 완료 후 기존 상태로 복구된다.
- [ ] 변환 중 `Ctrl+Enter`, 메뉴의 파일 추가/폴더 선택/삭제, 네이티브 드롭이 새 작업이나 스캔을 시작하지 않는다.

## 변환

- [ ] HWP -> PDF 1건이 성공하고 결과 PDF가 0바이트보다 크다.
- [ ] HWPX -> PDF 1건이 성공하고 결과 PDF가 0바이트보다 크다.
- [ ] HWP -> DOCX 1건이 성공한다.
- [ ] HWPX -> HWP 1건이 성공한다.
- [ ] 이미지 형식(PNG/JPG/BMP/GIF) 중 1개 이상을 변환하고, 현재 한글 버전에서 생성되는 출력 형태를 기록한다.
- [ ] HTML 또는 이미지 변환에서 기본 파일 외 보조 산출물이 생기는 경우 결과 JSON의 `created_files`에 기록된다.
- [ ] 동일 형식 변환(HWP -> HWP 또는 HWPX -> HWPX)은 변환 없이 `건너뜀`으로 표시된다.

## 안전장치

- [ ] 변환 전 `backup/` 폴더에 원본 백업이 생성된다.
- [ ] 원본 백업 옵션을 끄면 `backup/` 파일이 새로 생성되지 않는다.
- [ ] 기존 `backup/` 폴더가 있는 폴더를 변환해도 하위 백업 파일은 변환 대상에 포함되지 않는다.
- [ ] 출력 파일명이 충돌하면 번호 또는 타임스탬프가 붙은 새 이름으로 저장된다.
- [ ] PNG/HTML 등 보조 산출물이 이미 있는 경우에도 덮어쓰기 해제 상태에서는 새 이름으로 저장된다.
- [ ] `기존 파일 덮어쓰기`가 켜져 있어도 같은 실행 배치 안의 중복 출력 경로는 새 이름으로 분리된다.
- [ ] 기존 출력 파일이 갱신되지 않은 경우 기존 파일 존재만으로 성공 처리되지 않는다.
- [ ] 실패 자동 재시도 횟수를 0~3으로 바꿀 수 있다.

## 결과와 종료

- [ ] 사전 점검 다이얼로그에 실행 대상, 건너뜀, 출력 경로, 백업 설정, 입력/출력 차단 오류, HWP COM ProgID 상태가 표시된다.
- [ ] 결과 다이얼로그에 성공/실패/건너뜀/취소됨 집계가 표시된다.
- [ ] 결과 CSV 저장 시 `retry_count`, `backup_file`, `backup_error`, `created_files`, `output_size`, `output_mtime`, `save_format`, `progid_used` 열이 포함된다.
- [ ] 결과 JSON 저장 시 `created_files`가 배열로 저장되고 동일 감사 필드가 포함된다.
- [ ] 결과 TXT/CSV/JSON 저장 중 실패해도 `.tmp` 파일이 남거나 부분 파일이 최종 결과로 보이지 않는다.
- [ ] 보안 모듈 등록 실패 또는 앱 소유 PID 추적 실패 시 결과 경고에 표시된다.
- [ ] 실패 목록 TXT 저장이 동작한다.
- [ ] 변환 중 취소가 동작하고 남은 항목이 `취소됨`으로 표시된다.
- [ ] 무응답 상태에서 강제 종료는 앱이 띄운 한글 프로세스에만 적용된다.
- [ ] 변환 중 창 닫기 시 취소/종료 대기 흐름이 동작한다.
- [ ] 실패가 포함된 작업 완료 후 상태바의 한글 상태가 즉시 정상 대기로 덮이지 않는다.
